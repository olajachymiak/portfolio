About
Who I am

I'm a certified Product Owner & Scrum Master at Jacobs in Kraków, owning product delivery for internal tools across a multi-team programme. My background in financial services, document controlling, and AI‑powered product development means I bring both the technical fluency to build and the commercial awareness to prioritise.

But I don't stop at roadmaps and ceremonies. When I see a gap — a process that wastes hours, a workflow that frustrates people — I build the fix. Six automation systems, all driven by own initiative.
None of them assigned.

Work I'm proud of
Six systems. One common thread.

Someone mentioned a problem. I built a solution.

Resource Request System
POs had no visibility into resource assignments — they relied on ad-hoc Teams messages and emails to the resource manager. I replaced the manual MS Forms + Excel flow with a custom Azure DevOps system: typed work items, automatic epic linking, a dedicated board, and real-time Teams notifications via webhooks.
→ 20+ person programme — zero chasing, full transparency
Azure DevOpsPower AutomateWebhooksMS Teams
Automated Sprint Dashboard
Developers were searching through multiple Excel files to find their sprint assignments. I built an end-to-end pipeline: Power Automate triggers every two weeks, pulls allocation data from Excel, pushes to Azure repo, fires a Python pipeline, deploys a formatted summary to DevOps Wiki.
→ Single source of truth — updated automatically, no manual effort
PythonAzure PipelinesPower AutomateDevOps Wiki
Quality Reporting Automation
The quality team was sending ~300 emails manually every month — filtering, sorting, doing VLOOKUPs. I built a pipeline: new file on SharePoint triggers Power Automate → Azure Blob → custom Python/Pandas function → personalised emails, consolidated per recipient. Runs automatically, even when the team is away.
→ ~300 manual emails per month → zero
PythonPandasAzure FunctionsSharePoint
Cross-Reference Checker
Specs processors were manually Ctrl+F-ing through 20–200 files per project to validate section references against the table of contents. I wrote a standalone Python .exe they could run locally — select a folder, get a clean Excel summary report.
→ 1,000+ hours saved to date — team came back for a second tool
PythonDesktop AppExcel Output
Bug & Support Triage System
Bugs were reported via MS Forms, then manually reviewed in a private Excel. Nothing was visible to leadership, and submissions could get lost. I connected the form to Azure DevOps via Power Automate — auto-creating typed work items with auto-assignment, plus a SharePoint tracker with direct links to each DevOps item.
→ Full leadership visibility — no submission ever lost
Azure DevOpsPower AutomateSharePointMS Forms
GitHub PR Notifications
Developers relied on manual pinging and checking Outlook to know when a pull request was ready. I set up GitHub webhooks connected to Power Automate — a Teams bot now posts real-time alerts with branch name, author, and a direct link on every new PR and merge to main.
→ Zero manual pinging — instant awareness across the team
